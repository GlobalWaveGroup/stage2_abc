## 一、核心哲学

行情的本质是不同级别的延续和震荡的分形递归的复合体。

市场在每个瞬间同时在所有尺度上运动。任何行情都可以在某个尺度上被表达为三波段结构。
交易员的判断本质上就是：看到前两段(X,Y)，预测第三段(Z)。

## 二、核心概念定义

### 2.1 zigzag(2,1,1) — 原子级度量工具

- depth=2, deviation=1点, backstep=1
- 最少过滤，捕获每一个微摆动
- 它是起点原料，不是信号，不是目的

### 2.2 波段(Segment)

zigzag拐点之间的一段价格运动。每个波段有一个状态向量：
- 幅度（价格变化的绝对值）
- 时间（K线数量）
- 模长（价格路径的实际长度，非首尾直线距离）
- 角度/斜率
- 内部结构复杂度（内部包含多少更细粒度的波段）
- 其他待发现的维度

### 2.3 归并(Merge) — 多视野结构生成器

从zigzag(2,1,1)的原始拐点出发，通过归并生成不同尺度的波段。
归并是生成手段，归并级别不是波段的属性标签。
归并完成后，所有波段平等地进入波段池。

#### 2.3.1 幅度归并（已实现，MT5指标已有）

规则：相邻三拐点(P1,P2,P3)，如果P3超过P1（趋势吞没），合并。
- 多轮贪心执行
- 具有唯一性，可作为锚点
- 擅长识别趋势延伸结构

#### 2.3.2 时间/横向归并（待实现）

规则：幅度归并执行完毕后的剩余序列中，所有不满足幅度归并条件的
相邻三波段，进行贪心合并。

逻辑推导：
- 幅度归并拒绝的 = P3没有超过P1 = 价格在收敛/横向发展
- 横向归并继续吞入后续波段，直到某个新点突破了横向段的价格范围
- 突破点就是新趋势段的开始

产生的新线段类型：横向延续段（时间长、价格收敛）
这种线段幅度归并永远无法生成。

擅长识别的模式：
- 极端趋势 + 长时间横盘（趋势时间极短幅度极大 vs 横盘时间几十倍）
- 交易员眼中的"整理形态"

#### 2.3.3 冗余清理（待设计）

两种归并可能产生几何级数量的线段。
需要一个删减逻辑，去掉无代表性和重要性不高的线段，减少冗余信息。
具体规则待定，可能需要先看到归并结果后再设计。

### 2.4 波段池(Segment Pool)

所有归并（幅度+时间）产生的所有波段，不区分来源，平等混合。
包含：趋势段、调整段、横向延续段，所有尺度。

### 2.5 可比性(Comparability)

波段池中，两个波段(X,Y)是否可以配对：
- 相邻且方向相反
- 各特征维度上的关系落在可比性边界内
- 可比性是连续量，不是二元判断
- 不受归并级别限制：不同尺度的波段可以跨级配对，只要可比性成立

### 2.6 关系向量 R(X,Y)

一个可比的(X,Y)对的多维关系描述：
- 不是单一的 B/A 回撤比
- 而是同时在幅度、时间、模长、角度等多个维度上的综合比值
- R 构成一个多维连续空间

### 2.7 X+Y -> Z 映射

核心问题：R(X,Y) 如何映射到 Z 的发展？

- Z 的条件分布 P(Z|R)？
- Z 的聚类模式？
- 连续映射函数？
- 以上哪种，不预设，让数据说话

泛化说明：
- 不局限于"A趋势B调整C延续"的叙事
- X+Y可以是任意相邻反向波段对（趋势+调整、趋势+横盘、调整+趋势...）
- 所有三波段结构都是合法的研究对象

### 2.8 多层次Z同时存在

在任意时刻t，波段池中存在多个可比的(X,Y)对（因为不同尺度的波段可以配对）。
每个对产生一个Z的预测。这些预测天然多层次同时存在。
这和市场本身的性质一致：市场在每个瞬间同时在所有尺度上运动。

### 2.9 策略向量

C（Z的初始预测）基于"C约等于A（X）"的假设，在多维度上用X的特征作为初始估计。
由此推导初始策略向量（入场、方向、大小、止损、止盈）。
随着实际C'（Z'）的展开，计算C'与预测C之间的离差，实时调整策略向量。

## 八、完整归并引擎设计（2026-02-25 v2.1确认）

### 8.1 核心哲学

道生一，一生二，二生三，三生万物。
任何三波结构，只要满足归并条件（幅度增加或时间增加），就必须归并。
幅度归并和横向归并是两种不同的"看"法，二者双向互生，交替推进。

### 8.2 总体架构：交替迭代 + 滑窗收集

```
while(有变化) {
    // 尝试幅度归并一轮
    if(幅度归并有变化) {
        滑窗收集：所有满足条件的三波都产出线段 → 波段池;
        同时：对当前序列做横向滑窗收集 → 波段池（中间级别的横向线段）;
        贪心推进：产生下一级拐点序列 → 快照;
        continue;  // 回到循环顶部
    }
    // 幅度无机会 → 尝试横向归并一轮
    if(横向归并有变化) {
        滑窗收集：所有满足条件的横向三波都产出线段 → 波段池;
        贪心推进：产生下一级拐点序列 → 快照;
        continue;  // 回馈，幅度归并再检查
    }
    // 两者都无变化 → 不动点
    break;
}
```

关键原则：
- 幅度优先：有幅度归并机会就先做
- 横向补充：幅度无法动时横向做一轮，结果回馈给幅度
- 双向互生：横向归并产生新锚点，可能创造新的幅度归并机会，反之亦然
- 滑窗不遗漏：每轮扫描中，贪心跳过的中间三波也通过滑窗收集进池
- 中间级别横向不遗漏：每轮幅度归并前的序列也做横向滑窗收集
- 所有快照和extra线段共存进入波段池，不删除任何信息

### 8.3 幅度归并规则

取相邻四拐点(P1,P2,P3,P4)：
- 如果P1和P4分别是四点中的最高和最低（极值对）→ 合并中间两点
- 贪心推进：从左到右扫描，找到就跳3步
- 滑窗收集：逐个检查所有4拐点组，满足条件的都记录（不跳跃）

### 8.4 横向归并规则

幅度归并无变化后的残留序列中，取相邻三段(a,b,c)，按几何结构分四类：

#### 类型1：收敛三角形 |a| > |b| > |c| → 连接
#### 类型2：扩张三角形 |a| < |b| < |c| → 连接
#### 类型3：b最长有交错 → 连接
#### 类型4：b最长无交错 → 不连接

交错判定：连接a起点到c终点的斜率
- 峰→谷线：slope <= 0 → 有交错
- 谷→峰线：slope >= 0 → 有交错

### 8.5 线段来源分类

波段池中的线段按来源标记：
- base: 基础ZG直接产生的相邻线段
- amp: 幅度归并贪心推进产生的快照线段
- lat: 横向归并贪心推进产生的快照线段
- amp_extra: 幅度归并滑窗收集的被贪心跳过的线段
- lat_extra: 中间级别横向归并滑窗收集的线段

所有来源平等进入波段池，自然去重。

### 8.6 完全归并（不动点）

迭代直到幅度归并和横向归并都无法产生任何变化。
实践中200根K线约9轮迭代，6级幅度+2批横向，229条extra线段。

### 8.7 性能参考（EURUSD H1, 200 bars, v2初始池）

- 基础ZG: 109拐点
- 不动点: 9轮迭代, 最终3拐点
- 幅度归并: 6级 (109→63→41→25→15→9→7)
- 横向归并: 2批 (7→5→3)
- 初始波段池: 301条去重线段 (base:108, amp:51, lat:2, extra:140)

## 九、v3: 波段池三波融合 — 消融级别界限（2026-02-25确认）

### 9.1 核心突破

**v2的问题**: 逐级归并的贪心吃掉重要中间拐点。如L4(bar39)在A5被消除，
导致 L4→H2(bar166) 永远无法产出。但池中已有 L4→H3, H3→L5, L5→H2 三条
首尾相连的线段。

**用户核心指令**: 线段就是线段，不分级别、不分来源。
任何三条首尾相连的线段 → 无条件产出新线段。
不需要幅度条件、不需要横向条件 — 这些作为后续冗余删除的判据之一。

### 9.2 pool_fusion() 算法

```
pool_fusion(pool, pivot_info):
    seg_set = pool转为字典(key=(bar_start, bar_end))
    
    while True:
        建邻接索引: end_at[bar], start_at[bar]
        
        搜索所有三波组合:
            for 每个枢轴点 p2:
                for seg_A 结束于 p2:
                    for seg_B 起始于 p2:
                        p3 = seg_B.end
                        for seg_C 起始于 p3:
                            new_seg = seg_A.start → seg_C.end
                            if 不存在于 seg_set:
                                加入
        
        if 无新线段产出:
            break  # 不动点
```

- F3: 不动点
- **总计: 2970条, 0.166秒**

### 10.3 对称结构识别（5种维度对称，2026-02-25确认）

在波段池中寻找三波(A,B,C)对称结构，5种维度:
1. **幅度对称**: |amp_A| ≈ |amp_C| — 价格空间等距
2. **时间对称**: time_A ≈ time_C — 时间空间等距
3. **模长对称（核心）**: mod_A ≈ mod_C — 时-空联合度量
   - mod = sqrt(amp² + time²)，几何含义：价格-时间直角三角形的斜边
   - **关键问题**: 幅度和时间归一化机制不同，直接计算模长会被时间淹没
   - 存在一个"时-空转换动态常数"使两者可比，**理论上存在但尚未推导**
   - 当前方案: 用经验归一化缩放先跑，记住这是待解的核心问题
4. **斜率对称**: slope_A ≈ -slope_C — 镜像倾斜角
5. **内部结构对称**: complexity_A ≈ complexity_C — 分形复杂度（内部子波段数）

注: 速率(amp/time)已删除，与斜率同质。

**核心价值**: 每个对称结构的结束位置 = 重要转折点预测

### 10.4 动态K线生命周期

- 每条线段: birth_bar → 发展 → death_bar
- 在K线推进过程中，多条线段同时存活
- **多线段对称均衡共振** → 转折概率大增
- 这个共振信号 → ML特征向量 → 机器学习预测转折

## 十一、几何形态分类（2026-02-25用户描述，2026-02-26录入）

### 11.1 核心原则

所有几何形态都可用3段或5段表达。对称性分两类：
- **镜像轴对称**: 以某条竖直线（通常是B段中点）为对称轴，A和C镜像
- **中心对称**: 以某个点（通常是B段中点）为旋转中心，A和C旋转180°对应

### 11.2 形态分类表（约10种典型）

#### 镜像轴对称类

| 形态 | 段数 | 结构 | 对称描述 |
|------|------|------|----------|
| V底/V顶反转 | 2段(AB) | 下跌A + 上涨B (或反向) | 以AB转折点为轴的镜像对称 |
| 双顶/双底 | 4段(ABCD) | 涨A + 跌B + 涨C + 跌D | 以B-C中间线为轴的镜像对称 |
| 头肩顶/底 | 6段(ABCDEF) | 三峰(三谷)结构 | 以中间峰(谷)为轴的镜像对称 |

#### 中心对称类

| 形态 | 段数 | 结构 | 对称描述 |
|------|------|------|----------|
| abc调整结构 | 3段 | 调整浪a + 反弹浪b + 调整浪c | 以b段中心点旋转对称，a≈c |
| 12345推动结构 | 5段 | 1推+2调+3推+4调+5推 | 以3段中心点旋转对称 |

### 11.3 关键发现

- 所有形态本质上都可用 **3段(abc)** 或 **5段(abcde)** 表达
- 3段是最小完整结构单元（两段确定一个关系，三段产生对称预期）
- 对称结构的结束位置 = 重要转折点的概率预测
- 同一段行情在不同尺度上同时满足多种对称 → 共振 → 转折概率最大化
